# Architecture

One page to orient anyone landing in this repo. Product intent lives in
`PRD.md`; this file maps how the code is shaped today.

## The one-paragraph picture

Codex Desktop runs the AI agent. This repo is the **harness around it**: a
zero-dependency Python CLI (`chat-lms` / `python -m chat_lms_agent`) that
Codex lifecycle hooks invoke on every session event. The harness injects
context, gates dangerous actions before they run, blocks session close until
knowledge is recorded, and keeps every ledger in a **private profile
workspace** that is physically separate from this public repo.

```text
Codex Desktop (the agent runtime — brain, chat UI, tool execution)
   │  lifecycle hooks (hooks/hooks.json; private copy generated by bootstrap)
   ▼
chat-lms CLI (this repo, src/chat_lms_agent/)
   │  reads/writes ONLY through sanctioned commands
   ▼
Private profile workspace (<LOCALAPPDATA>/…, never inside this repo)
   └─ .chat-lms-state/  approvals, memory, journals, goals, blocks, telemetry
```

## The two worlds (hard boundary)

| Public repo (this checkout) | Private profile workspace |
| --- | --- |
| Code, tests, plans, docs, repo-default route packs and model catalog | Learner data, DB, reports, backups, memory, every ledger |
| Safe to publish; privacy tests scan it | Created by `scripts/bootstrap.ps1`; self-syncs safe wiring each session |

`state.py resolve_profile_state` rejects any runtime state under the repo
(`PUBLIC_REPO_STATE_REJECTED`, exit 4). `journal.redact_runtime_text` runs
secret/path redaction then learner-PII pseudonymization (`privacy.py`) on
every model-bound text.

## Hook lifecycle (what fires when)

| Event | Module path | What happens |
| --- | --- | --- |
| SessionStart | `commands.py` → `context.build_host_context` | Full budgeted context: tools, memory (levels + budget), routes, model catalog, contracts. Claims compact-recovery if pending. |
| UserPromptSubmit | `context.build_prompt_delta_context` + `prompt_routes.py` / `route_packs.py` | Tiny delta payload: route card (built-in or data pack), pending approvals, top-K memory recall (`memory_recall.py`). |
| PreToolUse | `pre_tool_gate.py` | Tier decision table (`read/write/exec`): denies destructive-without-approval, runtime-ledger writes, private→public leaks; unknown tools → ask. Denials journaled. |
| PostToolUse | `commands.py` | Memory-obligation guard only (registry changes need `--memory-updated`); otherwise silent PASS. |
| PostCompact | `session_closeout.record_compact_event` | No stdout (host rejects it). Stores a recovery marker; the next SessionStart/prompt re-injects once. |
| Stop | `session_closeout.write_stop_closeout` | Native `decision:"block"` + Korean remediation while obligations/approvals/import plans remain; `stop_hook_active` no-op; 3rd identical block escalates to the teacher. |

Hook stdin is byte-read with UTF-8-first decoding (locale fallback) and a
1 MiB cap (`hook_payloads.py`). Payloads are accepted in the host dialect
**or** the neutral `harness-event-v1` envelope (`harness_events.py`).

## Governance ledgers (all under `.chat-lms-state/`)

| Ledger | Module | Contract |
| --- | --- | --- |
| `approvals.json` | `approvals.py`, `approval_handlers.py` | planned→approved→consumed, single-use; approve requires a real terminal + typed id; agent actor structurally rejected. |
| `trace/`, `audit/` | `journal.py` | Per-record JSON with schema versions; pre-tool denials and academy imports leave records. |
| `memory.json` (+ archive) | `state.py`, `memory_handlers.py`, `memory_obligations.py` | Changed-file classes derive required keys; draft → human apply; 2000-char write cap; levels control hydration. |
| `goals.json` | `goal_state.py` | Evidence-marker-gated verification with `qa_verifier_status`. |
| `agent-tool-lifecycle.json` | `agent_tool_lifecycle.py` | draft→registered→active→deprecated; promote needs `--evidence`; deprecated is terminal. |
| `block-lifecycle.json` + `side-panel-drafts/` | `side_panel_blocks.py` | Same transitions plus: quarantine drafts, preview validation, promote needs evidence + `panel:<id>` memory + consumable teacher approval, deprecate needs a closing report. |
| `usage-telemetry.json` | `usage_telemetry.py` | Surface ids/counts/timestamps only; feeds promotion *suggestions* at closeout. |
| `qa-ledger.jsonl` + `install-id` | `self_qa.py` | Consent-gated harness anomaly log; fixed fields (learner data cannot ride); bounded rotation. |
| `privacy.json` (+ reverse map) | `privacy.py` | Reversible placeholders restored only on owner surfaces; one-way entries never round-trip. |

## Extension surfaces (how the harness grows at runtime)

- **Agent tools** — proposal with five contracts → lifecycle pipeline; the
  composed read path (`tool_store.py`) merges static registry + legacy
  `tools.json` + lifecycle records so reuse-check (`agent_tool_reuse.py`)
  and hydration see everything.
- **Side-panel blocks** — the same pipeline for UI building blocks, with
  quarantine and teacher approval (see ledger table). Production lists never
  include drafts.
- **Prompt routes** — data packs under `routes/` (repo defaults) and the
  profile `routes/` dir; three buckets (always_inject / listed_lazy /
  trigger); one malformed pack warns, never aborts (`route_packs.py`).
- **Shortcut registry** — profile-only `shortcut-v1` files under
  `.chat-lms-state/shortcuts/`; `chat-lms shortcut` stores and replays
  user-registered commands through injectable command/browser seams.
- **Skills** — `.agents/skills/*/SKILL.md`, validated by `skills.py`.

## Academy data binding

Academy imports and panel readers share the canonical entity-field contract
documented in `docs/academy-data-contract.md`. Import accepts legacy
`learner_id`/`class_id` field names, normalizes them to canonical `id` fields
for persisted store data, and warns before approval when a learner lacks the
display `name` a reader needs.

## Natural-Language Panel Pipeline

Panel-style prompts go through a fixed pipeline instead of prompting the agent
to author ad-hoc HTML. `prompt_routes.py` resolves the teacher's natural
language against the built-in wordbook route and loaded route packs. If nothing
matches, `prompt_route_catalog.py` exposes route cards so the host model can map
the request to a known `route_id` and run the catalog `first_command`. Each
route lands on a fixed CLI surface that opens a user-owned viewer reading data
only through local read-only `/api/` endpoints; the product code owns the
payload contract, never the agent.

## Model & host independence

- `docs/model-catalog.json` + `model_catalog.py`: role → family → concrete
  (`main_model → opus → claude-opus-4-8`), teacher profile overrides, typed
  errors for dangling/deprecated aliases, `harness model
  resolve|list|validate` CLI, compact role chart in hydration.
- `hosts.py` is the **only** core module that may name the host (actor
  string, runtime label, workspace dirname, host files).
  `tests/test_host_independence.py` greps the rest of `src/` for host tokens
  and runs a fake-host end-to-end cycle through `harness-event-v1` only.

## CLI shape

`commands.py` dispatches ~18 namespaces (doctor, context, memory, session,
hook, approval, trace/audit, goal, agent-tools, side-panel, academy-db,
harness, skills, bootstrap, …) parsed by `command_parser.py` (+
`v3_command_parser.py`, `lifecycle_parser.py`, `academy_db_parser.py`).
Every command emits one-line JSON with a stable exit-code contract:
0 PASS · 2 ERROR · 3 NEEDS_APPROVAL · 4 UNSAFE · 5 BLOCKED.

## Invariants the test suite enforces

- Hydration payload is deterministic (sha-stable) and inside pinned byte
  ceilings imported from `context.py` (`tests/test_context_budget.py`).
- Core modules are host-token free (`tests/test_host_independence.py`).
- The repo contains no private paths/credentials
  (`tests/test_repo_privacy.py`), and tests run hermetically — ambient
  credentials blanked, profile redirected per test (`tests/conftest.py`).
- Hook exit codes are pinned per event; PostCompact emits nothing
  (`tests/test_hooks.py`).
- Docs are contracts (`tests/test_docs_contract.py`), including this file's
  existence and the README staying current.
